# Contexto de Documentación

Esta carpeta contiene la documentación formal del proyecto.

---

# Objetivos

- Mantener trazabilidad.
- Registrar arquitectura.
- Mantener requisitos claros.
- Documentar decisiones globales.
- Reflejar el estado real del proyecto.

---

# Reglas

- No generar documentación ficticia.
- Toda documentación debe representar decisiones reales.
- Mantener consistencia entre requisitos y arquitectura.
- Evitar duplicación innecesaria.

---

# Áreas

## requisitado/

- requisitos funcionales

- casos de uso
